phase_plan:
  planner_id: surface
  summary: >
    Сравнительный анализ коммуникационных модулей/стандартов для автономных
    транспортных средств (БПЛА/дроны/роботы) по 5+ критериям. Результат —
    структурированная Google-таблица с параметрами (тип, проводные/беспроводные,
    энергопотребление, Wi-Fi режимы, режимы связи, управление питанием/sleep,
    характеристики питания) и итоговая сводка для пользователя (2–3 ключевых
    вывода/слайда).

  phase_count: 4

  delivery_atmosphere:

    user_facing: таблица сравнения + короткое сообщение с ключевыми выводами

  assumptions:
    - Запрос относится к беспроводным/проводным коммуникационным стандартам
      для автономных систем (Wi-Fi, BLE, LoRa, NB-IoT, Zigbee, Cellular и др.)
    - "5 критериев/технологий" — 5 технологий сравниваются по набору параметров
    - Итоговый артефакт — Google Spreadsheet с развёрнутой таблицей
    - Финальное сообщение пользователю — 2–3 абзаца/пункта с ключевыми отличиями

  non_goals:
    - Глубокий аппаратный datasheet-анализ каждого чипа
    - Проектирование архитектуры системы питания
    - Рекомендации по покупке конкретных модулей

  phases:
    - id: p1
      name: Исходные данные и структура сравнения
      goal: >
        Определить 5 коммуникационных технологий/стандартов для автономных
        систем и зафиксировать полный набор параметров сравнения.
      logical_blocks:
        - block: Поиск и сбор характеристик по каждому стандарту
          details:
            - Wi-Fi (режимы: активный/хранитель/протокол управления)
            - BLE / Zigbee / LoRa / NB-IoT / Cellular (5 технологий)
            - Для каждого: тип связи, проводной/беспроводной, энергопотребление
            - Управление питанием: sleep-режимы, время пробуждения
            - Характеристики питания: напряжение, ток (активный/sleep)
        - block: Фиксация сырых данных в рабочей заметке
          details:
            - Каждая технология = одна запись с полями
            - Источники отмечаются для последующей верификации
      entry_condition: Запрос пользователя получен, тема определена
      exit_condition: >
        5 технологий × 7 параметров = 35 ячеек данных собраны и
        зафиксированы в текстовой заметке
      collapse_risk: medium
      must_persist_before_exit:
        - Список из 5 технологий с определениями
        - Полный набор параметров сравнения (7 столбцов)
        - Сырые значения для каждой пары технология×параметр
      context_notes: >
        Данные из веб-поиска могут быть противоречивы — фиксировать источник

    - id: p2
      name: Верификация и нормализация датасета

      logical_blocks:
        - block: Кросс-проверка значений

            - Проверить единицы (мА, мкА, мВт) — привести к единым
            - Уточнить sleep-режимы: deep sleep vs standby vs idle
        - block: Нормализация таблицы
          details:

      entry_condition: p1 завершён, сырые данные персистированы
      exit_condition: >

      collapse_risk: medium
      must_persist_before_exit:
        - Финальная нормализованная таблица в текстовом виде (все значения)
        - Список источников по каждой технологии
      context_notes: >
        Если данные из p1 уже остыли — потребуется recall из архива или
        повторный поиск. Проверять горячее окно перед верификацией.

    - id: p3
      name: Создание Google Spreadsheet — публичный артефакт

        Перенести верифицированный датасет в Google-таблицу с форматированием,

        - block: Структура таблицы

            - Столбцы: Тип | Проводной/Беспроводной | Энергопотребление (активный)
              | Энергопотребление (sleep) | Wi-Fi режим | Режим связи
              | Управление питанием | Характеристики питания | Use-case
            - Шапка с заморозкой первой строки
            - Условное форматирование: минимум/максимум по энергопотреблению
        - block: Заполнение и форматирование
          details:
            - Внести все верифицированные значения
            - Цветовое выделение: зелёный — низкое энергопотребление
            - Добавить лист "Источники" со ссылками
            - Настройка прав доступа (чтение/редактирование)
      entry_condition: p2 завершён, нормализованная таблица персистирована
      exit_condition: >
        Google Spreadsheet создан, все ячейки заполнены, ссылка доступна
      collapse_risk: low
      must_persist_before_exit:
        - URL Google Spreadsheet
        - Подтверждение количества заполненных ячеек
      context_notes: >
        Это точка персиста — после создания таблицы сырые данные в чате
        могут быть свернуты, но таблица остаётся долговечной.

    - id: p4
      name: Финальное сообщение пользователю
      goal: >
        Отправить пользователю краткое сообщение (2–3 ключевых вывода)

            - Ссылка на Google Spreadsheet

      exit_condition: >
        Пользователь получил сообщение со ссылкой и 2–3 выводами
      collapse_risk: low
